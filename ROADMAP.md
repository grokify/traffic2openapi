## v0.1.0 - Initial Release âœ…

### IR & Transport

- [x] Body capture with size limits
- [x] Header filtering (exclude sensitive headers)
- [x] Host filtering - Log only requests to specific hosts
- [x] Method filtering - Log only specific HTTP methods
- [x] Path filtering - Skip logging for specific paths
- [x] Request ID extraction from headers
- [x] Sampling - Log only X% of requests for high-traffic scenarios
- [x] Status code filtering - Skip logging for specific status codes
- [x] `AsyncNDJSONWriter` - Async file writer with error callback
- [x] `ChannelReader` - Read from `chan *IRRecord`
- [x] `ChannelWriter` - Write to `chan *IRRecord` for in-memory processing
- [x] `Engine.ProcessReader()` - Inference engine accepts `IRReader` for streaming
- [x] `Flush()` method for explicit buffer flushing
- [x] `GzipNDJSONReader` - Streaming gzip-compressed NDJSON reader
- [x] `GzipNDJSONWriter` - Compressed writer for gzip NDJSON output
- [x] `IRReader` interface for pluggable input sources
- [x] `IRWriter` interface for pluggable output destinations
- [x] `LoggingTransport` - http.RoundTripper implementation
- [x] `MultiWriter` - Fan out to multiple IRWriter destinations
- [x] `NDJSONReader` - Streaming NDJSON file reader
- [x] `NDJSONWriter` - Sync file writer
- [x] `SliceReader` - Read from existing `[]IRRecord` slice

### CLI Commands

- [x] Auto-reload on file changes
- [x] Breaking change detection - Detect breaking API changes
- [x] Compare OpenAPI specs - Show differences between two OpenAPI specs
- [x] Configurable debounce interval
- [x] Deduped view showing all parameter values
- [x] Deduplication for merge - Remove duplicate records by ID
- [x] Endpoint pages grouped by status code
- [x] Exit code for breaking changes - Non-zero exit on breaking changes for CI
- [x] Index page with method badges
- [x] JSON output for diff - Machine-readable diff output for CI/CD
- [x] Light/dark mode toggle
- [x] Merge IR files - Combine multiple NDJSON/JSON traffic files
- [x] Merge OpenAPI specs - Combine multiple OpenAPI specifications
- [x] Serve with Redoc
- [x] Serve with Swagger UI
- [x] Static HTML site generation - Generate HTML documentation from IR files
- [x] Watch mode for generate command - Regenerate on file changes

### Inference Engine

- [x] API key detection - Detect `X-API-Key` and similar headers
- [x] Basic auth detection - Detect `Authorization: Basic` headers
- [x] Bearer token detection - Detect `Authorization: Bearer` headers
- [x] Context-aware path parameter naming - `/users/{userId}` instead of `/users/{id}`
- [x] Cursor-based pagination detection - Detect `cursor`, `after`, `before`
- [x] Generate OpenAPI securitySchemes
- [x] Nested resource path parameters - `/users/{userId}/posts/{postId}`
- [x] Offset-based pagination detection - Detect `page`, `limit`, `offset`, `per_page`
- [x] Pagination parameter documentation - Add descriptions, defaults, constraints
- [x] Path pattern detection - Detect dates, filenames, slugs
- [x] Rate limit header documentation
- [x] Rate limit reset headers - `X-RateLimit-Reset`, `Retry-After`
- [x] Standard rate limit headers - `X-RateLimit-Limit`, `X-RateLimit-Remaining`

### Adapters

- [x] Playwright adapter for Python
- [x] Playwright adapter for TypeScript
